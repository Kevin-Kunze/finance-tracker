== Praktiken und Bewertung
//Describe or reference which management and technical practices will be used in the project, such as iterative development, continuous integration, independent testing and list any changes or particular configuration to the project. Specify how you will track progress in each practice. As an example, for iterative development the team may decide to use iteration assessments and iteration burndown reports and collect metrics such as velocity (completed work item points/ iteration).
Das Projekt wurde nach der Theorie der agilen Entwicklung umgesetzt. Wöchentlich am Montag war unser "Weekly", indem wir über die abgeschlossenen Tasks, Probleme und die Aufgaben der nächsten Woche besprochen und zugeteilt haben. Aller zwei Wochen hatten wir eine Retroperspective im Weekly, indem wir über generelle Probleme der Projektentwicklung gesprochen haben z.B. technische Aspekte wie auch Kommunikationsprobleme. Besonders am Anfang bei der Erstellung der Produktidee, User Stories, Personas und auch später für die Retroperspektive wurde Miro als Sammelboard sämtlicher wichtiger Informationen genutzt, damit das Team jederzeit vergangene Informationen nachvollziehen können. Für die Zuteilung der Aufgaben wurde ein Ticketboard im Github genutzt, indem der Progress jeder Aufgabe getrackt wird.
